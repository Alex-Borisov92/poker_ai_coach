# poker-ai-coach

Local post-session poker AI coach for a microstakes MTT player.

The app reads an already played Holdem Manager 3 SQLite `.hmdb` database in
read-only mode, builds local reports, and can send selected report or hand
context to an AI coach when AI is enabled.

The main AI Coach page uses a local tool-using agent. The model can request
bounded read-only tools such as HM3 aggregate stats, schema overview, stat
mappings, scout report, hand search, hand detail, tournament story, coaching
principles, and study drills. The backend executes those tools locally and sends
only bounded JSON or one selected historical hand text.

This is not a HUD, not a solver, and not a real-time poker assistant.

## Requirements

- Python 3.11+
- Node.js 20+
- A local HM3 `.hmdb` file

## Configure `.env`

Create `.env` from the example:

```powershell
Copy-Item .env.example .env
```

Put `.env` in the project root:

```text
<project-root>\.env
```

Minimal local config:

```powershell
HM3_DB_PATH=C:\path\to\database.hmdb
HERO_NAME=surok_valera
AI_ENABLED=false
OPENAI_API_KEY=
OPENAI_MODEL=gpt-5-mini
APP_HOST=127.0.0.1
APP_PORT=8000
```

Notes:

- `HM3_DB_PATH` must point to the local HM3 `.hmdb` file.
- You can also open Settings in the web UI and paste the `.hmdb` path for the current backend session.
- The Settings page does not upload or copy the database file.
- The backend shows only the database basename in API responses.
- Missing, wrong, or locked database files are reported as disconnected status.
- Keep `AI_ENABLED=false` until you are ready to send selected context to AI.

```powershell

```

Health check:

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/health
```

Database status:

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/database/status
Invoke-RestMethod http://127.0.0.1:8000/api/database/schema-overview
```

Create a sanitized schema snapshot for DBeaver:

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/database/explorer-snapshot -Method POST
```

Open the returned `local_exports\*_schema_snapshot.sqlite` file in DBeaver.
This snapshot is not a copy of the HM3 database. It contains schema metadata,
table counts, stat mappings, metric preview values, and agent tool names only.
It does not include full hand text, API keys, full source paths, or the original
`.hmdb` file.

Reports:

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/reports/overview
Invoke-RestMethod http://127.0.0.1:8000/api/reports/leak-finder
Invoke-RestMethod http://127.0.0.1:8000/api/study-plan
Invoke-RestMethod http://127.0.0.1:8000/api/tournaments
```

Training state:

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/training-runs -Method POST
Invoke-RestMethod http://127.0.0.1:8000/api/training-runs
Invoke-RestMethod http://127.0.0.1:8000/api/coach/deep-analysis -Method POST -ContentType "application/json" -Body '{"message":"Find deep leaks.","mode":"leak_finder_deep","limit":10}'
```

Training memory is stored in ignored local app state:

```text
local_state\coach.sqlite
```

This file is not the HM3 database. It stores local training runs, leak status,
study plan items, and chat messages.

Hand review:

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/hands/review-queue
Invoke-RestMethod http://127.0.0.1:8000/api/hands/1
```

## Enable AI

AI is disabled by default. To enable real AI calls:

```powershell
AI_ENABLED=true
OPENAI_API_KEY=your_api_key_here
OPENAI_MODEL=gpt-5-mini
```

AI safety behavior:

- No API key is hardcoded.
- Missing API key returns a clear disabled state.
- The full HM3 database is never uploaded.
- The AI Coach agent never receives the full `.hmdb` file.
- Agent tools run on the backend and use the same read-only SQLite helper.
- The agent can receive bounded report JSON, selected hand summaries, selected
  tournament story data, local coaching principles, or one selected historical
  hand text.
- Stats overview starts from HM3 aggregate stats in `compiledplayerresults`.
- Leak Finder and Study Plan are generated by the AI Coach agent from stats,
  safe context, local protocols, and selected hand IDs.
- Hand review sends one selected historical hand text and metadata only.
- Coach chat can use only selected overview, leak report, study plan, or one hand.

Coach agent example:

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/coach/agent-chat -Method POST -ContentType "application/json" -Body '{"message":"Run a database scout and find my top leaks.","session_id":null,"mode":"coach_overview"}'
```

## Troubleshooting

Missing DB path:

- Set `HM3_DB_PATH` in `.env`.
- Restart the backend after changing `.env`.

Wrong file:

- Use the HM3 `.hmdb` SQLite file, not a hand history text file.
- The app reports wrong file types without showing the full path.

Locked DB:

- Close HM3 import or maintenance tasks.
- Retry after HM3 releases the database lock.

Missing tables:

- The app keeps running and reports which expected HM3 tables are missing.
- Reports that need missing tables will show warnings or empty states.

Frontend cannot reach backend:

- Start FastAPI on `127.0.0.1:8000`.
- Check `VITE_API_BASE_URL` if you changed the backend port.
- Restart Vite after changing frontend environment variables.

## Docker Deployment

The Docker setup is meant for a safe cloud demo or private server deployment.
It does not copy your `.env` file or HM3 `.hmdb` database.

Build and run locally:

```powershell
docker compose up --build -d
```

Open:

```text
http://127.0.0.1
```

Cloud defaults:

- Backend runs inside Docker on port `8000`.
- Nginx serves the frontend on port `80`.
- Nginx proxies `/api/*` to the backend container.
- `AI_ENABLED=false`.
- `HM3_DB_PATH` is empty.
- Local app state is stored in Docker volumes, not in git.

Do not upload a real HM3 `.hmdb` file to a public cloud server unless you have
made a deliberate privacy decision. The recommended cloud demo mode is app only,
or app plus sanitized schema snapshot.

## GitHub CI, Releases, And Deploy

GitHub Actions are configured in `.github/workflows/`:

- `CI` runs backend tests, ruff, frontend typecheck, frontend build, Playwright
  smoke tests, and Docker build smoke on pushes and pull requests.
- `Docker Release` publishes Docker images to GitHub Container Registry and
  creates a GitHub release for tags like `v0.1.0`.
- `Deploy to Yandex VM` is a manual workflow that updates a VM over SSH.

Create a release from local git:

```powershell
git tag v0.1.0
git push origin v0.1.0
```

Or run `Docker Release` manually from GitHub Actions with a version input like:

```text
v0.1.0
```

Published images:

```text
ghcr.io/Alex-Borisov92/poker-ai-coach-backend:<version>
ghcr.io/Alex-Borisov92/poker-ai-coach-web:<version>
```

Required GitHub repository secrets for manual Yandex deploy:

```text
YC_HOST
YC_USER
YC_SSH_PRIVATE_KEY
YC_PORT
```

`YC_PORT` is optional if SSH uses port `22`. Do not store `.env`, API keys, or
HM3 database files in GitHub secrets unless you have made an explicit privacy
decision. The safe cloud default is app-only mode without a real HM3 database.

## Safety Notes

- Use this only for already played historical hands.
- Do not use it during active play.
- Do not connect it to poker clients.
- Do not scrape live tables.
- Do not treat it as a solver.
- Do not use it for real-time decisions.
- SQLite is opened read-only through `file:<absolute_path>?mode=ro`.
- Reports treat `1970-01-01` dates as unknown.
- Tournament filters also treat `1970-01-01` dates as unknown.

## Final Security Checklist

Before sharing, committing, or backing up the project:

- `.env` is not committed.
- API keys, tokens, cookies, and secrets are not committed.
- `.hmdb`, `.db`, `.sqlite`, and `.sqlite3` files are not committed.
- `local_state/` and `local_exports/` are not committed.
- Real hand histories are not committed.
- Generated personal reports are not committed unless explicitly intended.
- Screenshots with sensitive info are not committed.
- `frontend/node_modules/` and `frontend/dist/` are ignored.
- Tests use only tiny synthetic SQLite fixtures.
- Backend tests, ruff, and frontend build pass.
